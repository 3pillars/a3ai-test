# Brainstorming Skill (Adapted from Superpowers)

Use this before any implementation work to ensure clarity and alignment.

## When to Use

✅ **ACTIVATE when:**
- Building new features
- Adding functionality
- Creating components
- Modifying behavior

## Hard Gate

🚫 **DO NOT proceed to implementation until:**
1. You've explored the project context
2. You've asked clarifying questions
3. You've proposed 2-3 approaches with trade-offs
4. User has approved the design
5. Design doc is written and saved

## Checklist

1. **Explore context** — Check existing files, docs, recent changes
2. **Ask questions** — One at a time, understand purpose/constraints/success criteria
3. **Propose approaches** — 2-3 options with trade-offs + recommendation
4. **Present design** — Scaled to complexity, get user approval
5. **Write spec** — Save to `docs/YYYY-MM-DD-<topic>-design.md`
6. **Get user sign-off** — Before writing any code
7. **Move to implementation** — Then create plan

## Anti-Patterns

❌ "This is too simple to need a design"
❌ Jumping straight to code
❌ Implementing without user approval

## Simple Project Flow

For truly simple tasks (5 min or less):
1. Still ask: "What's the goal?"
2. State your approach
3. Get quick approval
4. Implement

---

**Remember:** A 5-minute design prevents 5 hours of wasted work.
