# Spec-Driven Development with Claude Code

Source: heeki.medium.com

## Key Concept
AI tools opened Pandora's box of "vibe coding" - need brakes to avoid tech debt. Spec-driven development enforces good practices.

## Three Levels of Spec-Driven Development

| Level | Description |
|-------|-------------|
| **Spec-first** | Write spec first, then use in AI workflow |
| **Spec-anchored** | Keep spec after completion for maintenance |
| **Spec-as-source** | Only edit spec, human never touches code directly |

## Author's Approach (Mostly Spec-first)

1. Read documentation first
2. Think through architecture
3. Break project into modular sub-projects
4. Define phases for each sub-project

## Example Workflow

```
1. Research phase → Read docs, understand requirements
2. Planning phase → Break into stacks/modules
3. Implementation → Claude Code executes
```

## Key Lessons

- **Plan first, code later** - Don't jump into generation
- **Modular stacks** - Easier testing and deployment
- **Feed spec back** - Continuous feedback loop
- **Think deeply** - Requirements drive better outputs

This aligns with Vasuman's lesson: "Architecture matters more than model selection" and Eyad's advice: "Think first, then type."
